Case Study

Image Sequence to Video Converter

A lightweight desktop application I built to solve my own workflow challenges when converting Blender animation sequences into video files with customizable frame rates.

Role

Product Designer & Developer

Timeline

1-2 Days

Tools

Python, PyQt, FFmpeg

Overview

This project is a desktop application that converts image sequences into video files with customizable frame rates. It was originally developed to streamline the process of exporting Blender animations (PNG sequences) into playable videos — eliminating the need for complex or paid tools like Adobe Premiere Pro.

The Challenge

3D artists and motion designers often export animations as image sequences (e.g., hundreds of PNGs) to maintain quality. However, converting these sequences into videos typically requires:

  • Opening a heavy video editor (Premiere, After Effects)
  • Adjusting settings manually
  • Exporting the video, which takes multiple steps

This process is time-consuming, resource-heavy, and unnecessarily complex for a simple task.

How might we create a lightweight, easy-to-use desktop tool that allows users to convert image sequences into videos quickly and without confusion?

My Process & Needs

As a Blender artist and animator myself, I often needed to convert animation sequences into videos. The existing solutions were either too complex or required expensive software.

My Key Requirements

  • Speed — I wanted a one-click solution to convert sequences without unnecessary steps
  • Simplicity — No need for complex video editing features I didn't need
  • Control — Just the essential settings I actually use (FPS, output format)

My Design Goals

Minimal Clicks

Select folder and hit convert - that's it

Smart Defaults

Pre-set to the settings I use most often

Progress Visibility

Clear indication of conversion progress

Design Process

1. Wireframing

I started by sketching a compact layout:

  • A clear "Browse Image Folder" button for easy navigation
  • Input fields for Frame Rate (FPS) and Output Format
  • A single prominent CTA ("Convert Now") to reduce confusion
  • A progress bar for user feedback and completion tracking

2. Visual Design

  • High contrast purple CTA to draw attention
  • Rounded corners and light-gray background for modern, tool-like feel
  • Simple top bar to make the app look professional and uncluttered

3. Interaction Design

  • Immediate feedback when selecting a folder or starting conversion
  • Progress bar animates smoothly to convey responsiveness
  • "Convert Now" button disabled during conversion to prevent accidental clicks

Key Features

Folder Selection

Easily select folders containing image sequences

Customizable Frame Rate

Adjust FPS to match your animation needs

Multiple Output Formats

Support for MP4, AVI, and more

Real-time Progress

Visual feedback during conversion

UX Decisions

Challenge UX Solution
Too many settings confuse users Minimal input fields — only essential controls
Users want to know progress Added a progress bar with % indicator
Default values unclear Pre-filled frame rate (30 FPS) as an industry standard
Ambiguity in next steps Single, bold "Convert Now" button with clear labeling

Personal Impact

Since building this tool, it's become an essential part of my animation workflow. What used to be a multi-step process that required opening heavy software is now just a couple of clicks.

How It Helps Me

  • Saves me several minutes per animation export
  • Eliminates the need to open resource-heavy video editors
  • Gives me exactly the control I need without overwhelming options

Future Improvements I'm Considering

  • Adding drag-and-drop support for even faster workflow
  • Remembering my last used settings and folder
  • Support for batch processing multiple sequences at once

The Result

What started as a simple script to solve my own problem has become an essential tool in my animation workflow. It's a perfect example of how identifying a personal pain point can lead to an elegant, focused solution.

While I initially built this just for myself, I've since shared it with a few fellow animators who found it equally useful. It's incredibly satisfying to see how a tool created out of personal necessity can benefit others in the community.
